# 闪念胶囊首页重构设计文档（蓝粉白·策展墙）

**日期**: 2026-03-19  
**项目**: flash-pill  
**范围**: 首页（`/`）布局与 UI 优化（不改后端数据结构）

---

## 1. 目标与边界

### 1.1 目标

1. 将首页视觉语言统一到蓝粉白设计稿，替换当前海盐绿主调。
2. 将首页信息重心从「输入 + 统计」切换为「卡片展陈优先」。
3. 在不改变核心业务流程（创建胶囊、查看胶囊详情）的前提下，提升内容浏览沉浸感与视觉层级。
4. 保持移动端可用性与性能稳定性，不引入重型动画依赖。

### 1.2 非目标

1. 不调整数据库 schema 与服务层接口（`getAllCapsules` / `createCapsule`）。
2. 不新增复杂搜索系统，仅保留轻量筛选交互入口。
3. 不重做详情页（`/capsule/$id`）信息结构，本轮仅要求视觉一致性可延续。

---

## 2. 信息架构（IA）

首页采用「策展墙」模式，首屏直接展示内容卡片。

### 2.1 区块优先级

1. **Primary**: 胶囊卡片墙（Bento / Masonry 混排）
2. **Secondary**: 悬浮式 Quick Capture 入口
3. **Tertiary**: 轻量统计信息（微型信息条）
4. **Optional**: 轻量筛选（状态 / 标签）

说明：本轮仅保留筛选入口的 UI 位置与视觉样式，筛选逻辑可延后，不阻塞主流程交付。

### 2.2 用户路径

1. 用户进入首页后先浏览卡片墙内容。
2. 用户点击卡片进入详情页（现有跳转逻辑保留）。
3. 用户在需要记录时使用悬浮 Quick Capture 创建新胶囊。
4. 创建成功后卡片墙更新并保持浏览上下文。

浏览上下文验收条件：创建成功后不发生路由跳转，且滚动位置偏差不超过 ±24px。

## 3. 布局系统与响应式规则

### 3.1 栅格规则

- `<640px`: 1 列
- `640px - 1023px`: 2 列
- `1024px - 1439px`: 3 列
- `>=1440px`: 4 列

间距规则：`gap-4`（移动）/ `md:gap-5`（中大屏）。

### 3.2 卡片尺寸策略

- **Large**: `2x2`（仅桌面断点启用）
- **Medium**: `1x2`
- **Small**: `1x1`

移动端统一降级为单列流式呈现，避免密度过高导致阅读断裂。

尺寸分配规则（用于实现与测试一致）：

1. `>=1024px`：按索引循环 `[Large, Small, Medium, Small, Small, Medium]`。
2. `640-1023px`：`Large` 自动降级为 `Medium`，其余不变。
3. `<640px`：全部降级为 `Small`。

实现方式约束：采用 CSS Grid + `col-span` / `row-span` 变体实现，不引入 JS masonry 计算布局。

### 3.3 区块落位

1. `QuickCaptureInput`
   - 桌面：顶部悬浮条（sticky，不遮挡内容）
   - 移动端：底部悬浮入口，点击后展开为底部抽屉输入
2. 统计信息：移出主网格，作为页面角落微型信息条（例如「本周 +12」）
3. 筛选入口：
   - 桌面：页面右上轻量 chips
   - 移动：底部 sheet 打开筛选

筛选范围约束：本轮仅实现筛选面板开合、选项高亮与交互动效；不实现基于状态/标签的数据过滤逻辑。

## 4. 视觉系统（蓝粉白）

### 4.1 颜色角色

- **画布白（Base）**: 页面主背景与大面积留白
- **结构蓝（Structure）**: 信息层级、链接、状态主信号
- **情绪粉（Accent）**: 重点提示、交互强调、渐变辅助

状态色映射：

- `SEALED`: 蓝
- `FERMENTING`: 粉
- `OPENED`: 紫蓝
- `ARCHIVED`: 中灰

要求：状态色仅用于点状/细条/徽标，不做大面积整卡铺色。

### 4.2 背景与层次

1. 白色基底 + 双径向轻雾（蓝、粉）
2. 极弱纹理层（网格或颗粒）增强空间感
3. 保持低对比背景，确保文字与卡片是主视觉焦点

### 4.3 卡片材质

1. 半透明浅玻璃白（`rgba(255,255,255,0.72~0.86)`）
2. 细边框（冷色低饱和）
3. hover 时边框过渡到蓝粉混合高光
4. 不使用夸张发光或大面积阴影，保持轻量高级感

### 4.4 字体与可读性

1. 标题层级：高对比、较强识别
2. 正文层级：优先阅读效率
3. 元信息：降低视觉权重但满足可见性
4. 所有正文文本满足 WCAG 2.1 AA：正文最小对比度 `4.5:1`，大字号文本 `3:1`

## 6. 交互与动效规范

1. **Hover**: `translateY(-2px)` + 阴影增强 + 边框高光，时长 `180-220ms`。
2. **Press**: 轻微收紧阴影反馈，避免布局位移。
3. **入场动效**: 首次加载分组错峰淡入（约 40ms 级联）。
4. **筛选切换**: 仅针对筛选面板开合与选项高亮做透明度过渡，不触发数据过滤重排。
5. **性能与无障碍**: 支持 `prefers-reduced-motion`，降级为透明度或无动效。

`prefers-reduced-motion` 降级细则：关闭位移动效与级联延迟，仅保留 120ms 透明度变化。

---

## 7. 数据流与状态处理

### 7.1 数据流

1. 首页通过 `useQuery(['capsules'])` 拉取胶囊列表。
2. Quick Capture 触发 `createCapsule` 创建。
3. 创建成功后 `invalidateQueries(['capsules'])`，列表刷新。

### 7.2 状态处理

- **空状态**: 使用 2-3 张占位引导卡（而非纯文本），保持版式一致。
- **加载状态**: 骨架或占位块与真实卡片尺寸一致。
- **错误状态**: 顶部轻提示 + 重试入口，不中断已渲染内容区。

---

## 8. 错误处理与退化策略

1. `createCapsule` 失败：保留用户输入内容并提供可重试提示。
2. `getAllCapsules` 失败：展示轻量错误条与「重试」按钮。
3. 低性能设备：自动降级动效（通过 `prefers-reduced-motion` 与低强度阴影策略）。
4. 超长内容：卡片内文本截断 + 详情页查看完整内容，避免撑破网格。

### 9.1 功能验收

2. 卡片点击可正确进入详情页。

### 9.2 视觉与响应式验收

2. 无横向滚动。
3. 文本对比度符合可读性要求。
4. 浅色与深色模式边框/背景均可区分。
